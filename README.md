# clipmuxd

A featherweight phone ↔ PC bridge. No Electron. No account. No lag.

Send text, links, images, videos, and any file between your Android phone and Windows PC — instantly, privately, from anywhere.

## Why this exists

Using WhatsApp Desktop just to move a link or a photo between your phone and laptop burns 350–550 MB of RAM and can noticeably lag a heavily-loaded PC. Discord, Telegram Desktop, Slack — all the same. Electron apps are great products, but they're expensive bystanders when all you need is "put this text on the other screen."

clipmuxd is a single ~6 MB Go binary that sits in the background at ~10 MB RAM, does nothing until you use it, and replaces the WhatsApp-as-a-clipboard workflow entirely.

The name: a **clip**board **mux**ing **d**aemon — it multiplexes text, files, and media between your devices through one tiny endpoint.

## Features

- **Auto-clipboard.** Text sent from phone lands on your PC clipboard. Ctrl+V anywhere.
- **Images to clipboard, losslessly.** Send a photo, then paste it directly into Discord, Word, Paint, or any app that accepts image paste. Original bytes preserved via CF_HDROP file drop.
- **Any file, any size.** Streamed to disk, up to 5 GB per upload.
- **Manager UI** with tabs: Text, Links, Images, Videos, Audio, Docs, Archives, Other. Auto-categorized.
- **Thumbnail gallery** for images and videos. Tap to open full-size.
- **PWA.** Installs to your Android home screen as a one-tap app.
- **Android share target.** After install, "clipmuxd" appears in the system share sheet. Share a YouTube link, photo, or PDF straight to your PC.
- **Live updates** via Server-Sent Events. Inbox refreshes instantly when something arrives.
- **Works anywhere** — pair with free [Tailscale](https://tailscale.com) for encrypted access over mobile data, hotel Wi-Fi, anywhere.
- **Single shared token auth.** No login screen, no accounts, no cloud.
- **Zero runtime dependencies.** One `.exe`, all assets embedded. No npm, no Python, no .NET.

## Quick start

### 1. Build

Requires Go 1.22+.

```bash
go build -ldflags="-s -w" -o clipmuxd.exe ./cmd/clipmuxd
```

### 2. Run

```bash
.\clipmuxd.exe
```

It prints URLs like:

```
Open on your phone (same Wi-Fi):
  http://192.168.x.x:8080/?k=<auto-generated-token>
```

Allow the Windows Firewall prompt when it appears.

### 3. On your phone

Open the URL in Chrome, tap the menu, choose **Add to Home screen**. That icon is now your bridge to your PC.

## Remote access (any network)

To reach your PC when you are not on home Wi-Fi:

1. Install [Tailscale](https://tailscale.com/download) on PC and phone — free for personal use.
2. Sign in with the same account on both.
3. On your phone, use the PC's Tailscale IP (`100.x.y.z`) instead of the LAN IP:

   ```
   http://100.x.y.z:8080/?k=<token>
   ```

Traffic is WireGuard-encrypted end-to-end. Works on mobile data, café Wi-Fi, airports.

## Autostart on Windows login

Run once in PowerShell (no admin needed):

```powershell
$action = New-ScheduledTaskAction -Execute "$PWD\clipmuxd.exe"
$trigger = New-ScheduledTaskTrigger -AtLogOn -User $env:USERNAME
$principal = New-ScheduledTaskPrincipal -UserId $env:USERNAME -LogonType Interactive -RunLevel Limited
$settings = New-ScheduledTaskSettingsSet -AllowStartIfOnBatteries -DontStopIfGoingOnBatteries -StartWhenAvailable -ExecutionTimeLimit ([TimeSpan]::Zero) -RestartCount 3 -RestartInterval (New-TimeSpan -Minutes 1)
Register-ScheduledTask -TaskName 'clipmuxd' -Action $action -Trigger $trigger -Principal $principal -Settings $settings -Force
```

Remove later with:

```powershell
Unregister-ScheduledTask -TaskName clipmuxd -Confirm:$false
```

## Firewall (if phone can't reach PC on LAN)

Admin PowerShell, once:

```powershell
New-NetFirewallRule -DisplayName "clipmuxd" -Direction Inbound -Program "$PWD\clipmuxd.exe" -Action Allow -Profile Any
```

## Config flags

| Flag | Default | Description |
|---|---|---|
| `--addr` | `:8080` | Listen address |
| `--inbox` | `%USERPROFILE%\clipmuxd\inbox` | Where received files land |
| `--token` | auto-generated | Override auth token |
| `--url` | | Print URLs and exit |

The auth token is generated on first run and persisted at `%USERPROFILE%\clipmuxd\token`. Delete that file and restart the server to rotate it.

## Architecture

```
 ┌────────────────────┐         ┌─────────────────────────────┐
 │  Android Phone     │         │  Windows PC                 │
 │  (Chrome PWA)      │◀── HTTP ─▶│  clipmuxd.exe             │
 │                    │         │  ├── :8080 HTTP server      │
 │  • Paste box       │         │  ├── /api/send              │
 │  • File picker     │         │  ├── /api/inbox             │
 │  • Share target    │         │  ├── /api/ws   (SSE)        │
 │  • Live SSE        │         │  └── writes to inbox/       │
 └────────────────────┘         └─────────────────────────────┘
          │                                   │
          │ LAN: direct                       │
          │ Remote: via Tailscale tunnel      │
          └───────────────────────────────────┘
```

- **Server:** Go 1.22 stdlib `net/http`. Single binary. Web assets embedded via `//go:embed`.
- **Transport:** plain HTTP on LAN (same Wi-Fi) or via Tailscale's WireGuard tunnel (remote).
- **Auth:** 24-byte cryptographically random token in URL, also stored as a cookie after first load.
- **Live push:** Server-Sent Events (no WebSocket deps).
- **Clipboard integration:** Windows-only. Text via `clip.exe`; images via PowerShell + `System.Windows.Forms.Clipboard` (file drop + bitmap).

## Security model

What protects you:

- 24-byte random token required for every API request.
- Path traversal protection: filenames sanitized with `filepath.Base`, rejected if they contain `..`.
- Command injection closed: file paths are passed to PowerShell as parameter arguments, never interpolated into script text.
- 5 GB max body size per upload (`http.MaxBytesReader`).
- `HttpOnly` + `SameSite=Lax` auth cookie — blocks JS access and cross-site POST.
- Token lives in `%USERPROFILE%\clipmuxd\token` with `0o600` perms, outside the repo, not logged.
- No external network calls. No telemetry. Nothing leaves your PC.

What you should know:

- Over LAN, traffic is plain HTTP. Fine on your home Wi-Fi; not fine on public networks — use Tailscale.
- Anyone who sees the token URL can access your inbox. Don't share screenshots of your URL bar.
- The server only exposes the configured inbox directory. It does not serve other folders on your PC.

See [SECURITY.md](SECURITY.md) for the full threat model.
